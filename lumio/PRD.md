## 一、产品概述

### 背景与痛点

用户在微信生态中大量阅读公众号文章，同时也会在其他平台（小红书、微博、网页等）看到有价值的内容。现有收藏方式存在明显缺陷：

- **微信收藏**：需手动打标签，维护成本高，检索能力弱
- **文件传输助手**：消息流式结构，历史内容快速沉没，难以回顾
- **跨平台内容孤岛**：各平台内容无法统一管理

核心问题不是「存不进去」，而是「**收藏了就等于忘记了**」。

### 产品定位

> 生长在微信里的第二大脑——收藏无摩擦，知识会主动找你

**目标用户**：有内容收藏习惯、希望沉淀个人知识的微信重度用户（C端大众市场）

**技术形态**：微信服务号（收藏入口 + 消息推送）+ 微信小程序（主界面）

## 二、竞品分析

| 产品 | 优势 | 劣势 |
|------|------|------|
| Cubox | AI摘要强，多平台同步 | 需跳出微信，付费门槛高，无召回机制 |
| flomo | 轻量零门槛，服务号直记 | 不擅长处理外部链接，无AI能力 |
| Pocketify | UI精美，合集标签管理 | iOS专属，脱离微信生态，无AI，无召回 |
| 微信原生收藏 | 零门槛最无缝 | 无AI，无结构化，无提醒 |

**差异化核心**：三个竞品普遍缺失的能力恰好是本产品核心：
1. 微信生态原生零摩擦
2. AI自动摘要
3. 智能召回提醒

## 三、核心功能设计

### 3.1 用户核心流程

```
【收藏流程】
用户在微信看到文章
    │
    ▼
点击「分享」→ 选择「拾光」服务号 / 小程序
    │
    ▼
后台自动解析：标题 + 封面 + 正文 + AI摘要 + 关键词
    │
    ├─ 解析成功 → 服务号回复：「已收藏 ✓ 《文章标题》摘要：...」
    └─ 解析失败 → 服务号回复：「已保存链接，暂时无法解析内容」

【沉淀 / 主动查阅流程】
用户打开小程序
    │
    ├─ 首页：最近收藏（卡片流）
    ├─ 搜索：输入关键词 / 语义描述
    └─ 分类浏览：按话题 / 时间 / 已读未读

【召回流程】
后台定时任务（每天跑一次）
    │
    ▼
根据用户收藏时间 + 阅读行为，选出 1 篇召回文章
    │
    ▼
服务号推送：
「3周前你收藏了这篇文章，还没来得及看~
  《为什么大多数人做不到早起》
  点击回顾 →」
    │
    ▼
用户点击 → 跳转小程序阅读页
    │
    ├─ 标记已读 → 不再召回
    └─ 再次忽略 → 30天后再试一次，之后归入「永久库」
```

### 3.2 关键体验节点

| 时间点 | 事件 | 用户感知 |
|--------|------|----------|
| 第1次使用 | 分享文章 → 立刻收到AI摘要 | 惊艳时刻 |
| 第3天 | 收到第一条召回推送 | 建立信任 |
| 第30天 | 语义搜索找到旧文章 | 价值感知高峰 |

## 四、MVP 功能优先级

### Must Have（第一版必做）

**1. 收藏入口**
- 服务号接收转发链接，自动解析标题 + 封面 + 摘要
- 支持：微信公众号文章、通用网页链接
- 解析失败时保留原始链接，不暴露给用户

**2. AI 摘要生成**
- 转发后自动生成 200 字内摘要
- 提取 3-5 个关键词（用于后续召回和检索）
- 是第一版核心价值感知点

**3. 小程序主界面**
- 内容列表（卡片式，含封面 + 摘要预览）
- 基础关键词搜索
- 简单分类（未读 / 已读 / 全部）

**4. 智能召回（基础版）**
- 服务号每日推送 1 篇「今日回顾」
- 初版规则：从7天前未读内容中随机选取

### Should Have（第二版）

- 语义检索（自然语言找文章）
- 自动打标签/分类
- 阅读进度记录
- 图片、视频类内容收藏

### Nice to Have（后期）

- 知识图谱可视化
- 多人共享知识库
- 浏览器插件

## 五、智能召回机制

### 召回策略分层

**基础层：时间衰减召回**（类艾宾浩斯，但产品层不提这个名词）

```
收藏后第 3 天  → 第一次提醒（趁热打铁）
收藏后第 7 天  → 第二次提醒（加深印象）
收藏后第 30 天 → 第三次提醒（长期沉淀）
之后进入「永久库」，低频随机召回
```

**智能层：行为信号触发**
- 用户打开小程序但没看完 → 下次推「你还有篇没看完的文章」
- 某类话题近期收藏多 → 推「你最近对XX话题很感兴趣，这是一个月前的相关收藏」
- 季节/节点触发 → 收藏了年终总结相关文章，年底自动唤起

### 防骚扰设计

- 每天最多推 1 条
- 用户可设置推送时间窗（通勤 / 午休 / 睡前）
- 一键调节频率（正好 / 推少点 / 关闭）
- 推送文案有温度，不像系统通知

**文案示例：**
- ❌ 「您有 1 篇文章未读」
- ✅ 「3 周前你收藏了一篇关于职场的文章，还没来得及看」

## 六、商业模式

### 免费版

| 功能 | 限制 |
|------|------|
| 收藏文章 | 每月 30 篇 |
| AI 摘要 | 有，字数限制 |
| 基础搜索 | 关键词搜索 |
| 智能召回 | 每周 1 次 |
| 内容保存时长 | 6 个月 |

### 付费版（建议 ¥18-28/月 或 ¥98/年）

| 功能 | 说明 |
|------|------|
| 无限收藏 | 核心解锁点 |
| 完整AI摘要 + 关键词提取 | 更深度分析 |
| 语义检索 | 自然语言找文章 |
| 召回频率自定义 | 每天/每周自选 |
| 内容永久保存 | 不过期 |
| 自动分类/标签 | AI归类 |

### 变现路径

```
冷启动期 → 全功能免费，积累种子用户 + 口碑
成长期   → 开放付费订阅，免费版加限制
成熟期   → 团队版（企业知识库）作为第二曲线
```

> 采用订阅制而非买断：AI调用有成本，用户越活跃成本越高，订阅制更健康可持续。

## 七、技术选型

### 整体架构

```
┌─────────────────────────────────────┐
│           微信生态层                  │
│  服务号（收藏入口 + 消息推送）          │
│  小程序（主界面 + 阅读 + 搜索）        │
└──────────────┬──────────────────────┘
               │ 微信官方 API
┌──────────────▼──────────────────────┐
│           后端服务层                  │
│  Python（FastAPI）/ Node.js          │
│  内容解析 + AI摘要 + 召回调度          │
└──────┬───────────────┬──────────────┘
       │               │
┌──────▼──────┐  ┌─────▼──────────────┐
│   数据库     │  │    AI 服务          │
│  MySQL      │  │  DeepSeek API       │
│  + Redis    │  │  向量数据库(语义搜索) │
└─────────────┘  └────────────────────┘
```

### 技术栈明细

| 层级 | 选型 | 理由 |
|------|------|------|
| 小程序 | uni-app | 一套代码，后续可扩展其他平台 |
| 后端 | Python + FastAPI | AI调用生态最成熟 |
| 主数据库 | MySQL | 用户数据、文章数据 |
| 缓存/队列 | Redis | 召回队列、热数据缓存 |
| 向量数据库 | pgvector（MVP阶段）→ Milvus（成熟期） | 语义搜索 |
| AI摘要 | DeepSeek API | 国内合规，中文效果好，约¥1/百万tokens |
| 语义向量 | 阿里/百度 embedding 模型 | 有免费额度 |
| 内容解析 | readability.js / mercury-parser | 通用网页内容提取 |
| 定时任务 | Celery + Redis | 召回任务调度 |
| 部署 | 腾讯云轻量服务器 | 国内，微信生态亲和 |

### 初期部署成本估算

```
服务器（腾讯云轻量）：  ¥50-100/月
数据库（小规格）：      ¥30-50/月
AI API（DeepSeek）：   按量付费，1000用户约 ¥20-50/月
域名 + SSL：           ¥60/年
────────────────────────────────────
启动成本约 ¥100-200/月
```
