## 一、产品概述

### 背景与痛点

每个人每天都在写文字——朋友圈、小红书、公众号、产品文案、邮件、播客脚本。但「写出有人设、有风格的文字」依然是少数人的能力：

- **普通用户**：想让朋友圈像某位作家一样耐读，或像某位脱口秀演员一样有梗，但自己写出来总是干巴巴。
- **内容创作者**：需要稳定的人设语气，但写久了自我重复，缺新鲜感；同时想做播客却卡在脚本和角色对话上。
- **品牌 / 营销**：想为不同账号、不同 KOL 适配不同语气，目前只能靠人肉模仿。
- **通用 AI 写作工具**：会写"通顺的话"，但写不出"某个具体人会说的话"——风格扁平、人设模糊、容易"AI 味"。

核心问题：**用户不缺写作工具，缺的是「指定人设的笔」。**

### 产品定位

> 一支可切换人设的 AI 笔——把你的草稿，变成"那个人"会写的样子

**目标用户**：

1. **日常创作者**：朋友圈 / 小红书 / 微博博主，想让文字更有辨识度
2. **内容职业者**：公众号作者、自媒体、文案、播客主理人
3. **品牌 / 运营**：需要为多账号、多 KOL 维持稳定语气
4. **播客新人**：有想法但写不出对谈脚本

**一句话描述**：选一个人物风格，把你的文字改写成「他/她会写的样子」；进阶可一键生成多人对谈播客脚本。

**技术形态**：Web App（MVP）→ 浏览器插件 + 移动端

## 二、竞品分析

| 产品 | 类型 | 优势 | 劣势 |
|------|------|------|------|
| ChatGPT / Claude | 通用对话 | 能力强、可 prompt 模仿 | 风格不稳定、需用户自己写长 prompt、无人物库 |
| Notion AI / Grammarly | 写作助手 | 嵌入工作流 | 只做润色 / 改语气，不做"人设化" |
| Character.ai | 角色对话 | 人物库丰富 | 聊天导向，不输出"成稿文字" |
| 各类 "AI 仿写" 小工具 | 模板化改写 | 上手快 | 风格僵硬、播客等复杂场景缺失 |
| NotebookLM 播客 | 一键播客 | 音频质量好 | 只支持双人对谈、不可指定具体人设、不可控 |

### 竞品空白与机会

「**指定人设 × 高质量改写 × 可延展到播客**」三者的交集目前没有产品占据。Resona 把"风格"作为一等公民，从单段改写一路打通到多人对谈，覆盖从普通用户到内容职业者的连续需求带。

## 三、核心功能设计

### 3.1 人物风格库（产品的灵魂）

预置三类风格人物，用户也可自建：

#### 类型 A：公开风格（基于公开作品 / 言论训练）
- 文字作家：村上春树、王小波、毛姆、迟子建…
- 内容创作者：李诞、半佛仙人、辉太郎、papi…
- 历史 / 思想人物：鲁迅、苏轼、Naval Ravikant…

> ⚠️ 合规：人物风格库以「风格学习」而非「身份冒充」为定位，输出时显式标注「以 XX 风格改写」，并允许人物本人申请下架。

#### 类型 B：原创人格（产品自研，无版权风险）
- "深夜电台主持人 · 林沉"：温柔、长句、留白多
- "互联网毒舌选手 · 阿K"：短句、抛梗、反转
- "理性投资人 · Vega"：克制、数据驱动、不情绪化
- …（持续扩充，作为差异化资产）

#### 类型 C：用户自建人格
用户上传 5-20 段样本文字（可来自自己历史推文、某账号公开内容）→ 系统抽取风格特征 → 生成「私有人格」。

**风格特征维度**（每个人物在这些轴上有具体值）：
- 句式长度 / 节奏
- 用词偏好（书面 / 口语 / 网络 / 古风）
- 情感强度
- 修辞偏好（比喻 / 反讽 / 排比 / 直白）
- 价值观与世界观锚点
- 标志性词汇 / 口头禅

### 3.2 核心流程一：文案改写 / 扩写

```
【文案改写者旅程】
粘贴 / 输入原始文字（草稿、要点、想法）
    │
    ▼
选择人物风格（搜索 / 收藏 / 最近使用）
    │
    ▼
选择动作：
  ├─ 改写（保留原意，换语气）
  ├─ 扩写（要点 → 成稿）
  ├─ 缩写（长文 → 短帖）
  └─ 续写（接着写下去）
    │
    ▼
（可选）调节滑块：克制 ←→ 张扬 / 文雅 ←→ 口语 / 严肃 ←→ 玩梗
    │
    ▼
生成 3 个候选 → 用户挑选 / 微调 / 对比原文 diff
    │
    ▼
导出：复制 / 一键发小红书 · 微博 · 公众号草稿
```

### 3.3 核心流程二：播客生成（高阶能力）

```
【播客生产者旅程】
输入：一篇文章 / 一组要点 / 一个话题
    │
    ▼
选择播客形态：
  ├─ 单人独白（选 1 个人物）
  ├─ 双人对谈（选 2 个人物，自动设定主 / 副）
  └─ 圆桌讨论（3-4 人，自动分配立场）
    │
    ▼
选择 / 编辑节目结构：
  开场钩子 → 核心议题 → 观点冲突 → 个人故事 → 收束金句
    │
    ▼
生成脚本（带角色名、台词、停顿 / 笑声 / 语气标记）
    │
    ▼
逐段编辑：
  ├─ 改某句台词
  ├─ 让某个人「再激进一点 / 再克制一点」
  └─ 替换某个人物
    │
    ▼
（增值）一键 TTS 合成音频（按人物配音色）→ 导出 mp3 + 字幕
```

### 3.4 功能模块

| 模块 | 说明 | 优先级 |
|------|------|--------|
| M1 人物库与搜索 | 预置 + 自建 + 收藏 | P0 |
| M2 改写引擎 | 改 / 扩 / 缩 / 续 + 风格滑块 | P0 |
| M3 候选对比 | 3 候选 + 与原文 diff | P0 |
| M4 播客脚本生成 | 单人 / 双人 / 圆桌 | P1 |
| M5 自建人格 | 样本上传 → 风格抽取 | P1 |
| M6 TTS 配音 | 人物音色映射、合成导出 | P2 |
| M7 浏览器插件 | 任意网页选中文本 → 改写 | P2 |
| M8 团队空间 | 品牌共享人物库与术语表 | P2 |

## 四、交互设计原则

1. **以「人物」为入口，不是以「功能」为入口**：首页就是人物墙，点人即写。
2. **样张胜过描述**：每个人物卡片直接展示一段「同一原文 → 该人物改写」的对照，所见即所得。
3. **可控的不确定性**：3 候选 + 风格滑块，让"AI 创作"变成"我在选"。
4. **明确的人格边界**：永远显示「以 XX 风格生成」标签，避免冒名误导。
5. **从单段到长稿到播客的连续路径**：同一份草稿可以一路升级，不需要换工具。

## 五、商业模式

```
免费层：
  - 每天 5 次改写
  - 仅原创人格 + 部分公开人格
  - 无播客生成

创作者会员（¥29 / 月，¥288 / 年）：
  - 无限次改写 / 扩写
  - 全量人物库
  - 播客脚本生成（每月 10 期）
  - 自建 3 个私有人格

专业版（¥99 / 月）：
  - 播客脚本无限 + TTS 音频合成（按时长计费）
  - 自建私有人格不限
  - 浏览器插件 + API
  - 团队空间（5 席）

按需付费（无会员也可买）：
  - 单期播客脚本：¥9
  - 单期播客脚本 + 配音：¥39
```

**设计思路**：让最高频的「改写」用低门槛把用户带进来，让最有差异化的「播客」承担付费转化与口碑传播。

## 六、风险与对策

| 风险 | 对策 |
|------|------|
| 公众人物风格的版权 / 名誉风险 | 显式标注「以 XX 风格改写」、提供本人申诉下架通道、优先建设原创人格 |
| 风格趋同 / "AI 味"重 | 风格特征向量化 + 多候选 + 风格滑块；定期 A/B 评估"可识别度" |
| 通用大模型替代 | 护城河在「人物库 + 评测体系 + 播客工作流」，而非模型本身 |
| 自建人格被滥用（仿冒他人） | 自建人格的样本来源校验 + 仅本人可见 / 团队可见 |
| 播客 TTS 成本高 | 默认导出脚本，音频按时长付费，控制毛利 |

## 七、技术选型（MVP）

- **前端**：Next.js + Tailwind
- **后端**：FastAPI / Node
- **模型层**：自有 Prompt 编排 + 多模型路由（GPT-4o / Claude / DeepSeek / 文心）
- **风格抽取**：Embedding + 风格特征分类器（句长、词频、修辞标签）
- **TTS**：第三方音色 API（火山 / MiniMax / ElevenLabs）+ 人物 → 音色映射表
- **存储**：PostgreSQL + 对象存储（音频）

## 八、核心指标

| 指标 | 目标 | 含义 |
|------|------|------|
| 改写完成率 | > 80% | 生成后用户复制 / 导出的比例 |
| 候选采纳分布 | 三选一无明显偏差 | 候选质量均衡 |
| 人物复用率 | > 40% | 用户 7 日内重复使用同一人物 |
| 播客转化率 | > 8% | 改写用户尝试播客功能的比例 |
| 付费转化率 | > 6% | 免费转付费 |
| 月留存 | > 35% | 创作工具的健康基线 |
