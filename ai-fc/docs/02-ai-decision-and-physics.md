> 头脑风暴第二轮：三层决策架构、物理模型、空间分区、团队AI

## 一、AI决策树：三层架构

AI球员的决策不是一颗巨大的树，而是**感知→思考→行动**三层流水线，每帧（或每tick）执行一轮。

---

### 第一层：感知层（球员"看到了什么"）

每个球员都有一个**感知范围**，由"视野"属性决定。这不是圆形范围，而是**扇形+距离衰减**：

```
                  视野扇角（如120°）
                ╱      |      ╲
               ╱  清晰  |  清晰  ╲
              ╱   区域  |   区域   ╲
             ╱          |          ╲
            ╱    模糊区域  |  模糊区域   ╲
           ╱              |              ╲
          ╱________________|________________╲
                     球员朝向 →
```

**感知输出（每帧生成一次）**：

| 感知项 | 数据结构 | 说明 |
|--------|----------|------|
| 球的位置 | `(x, y, z)` | 球在地面或空中 |
| 球的速度 | `(vx, vy)` | 球的运动方向和速度 |
| 队友位置 | `[(id, x, y, status)...]` | 队友在感知范围内的才返回 |
| 对手位置 | `[(id, x, y, threat_level)...]` | 对手在感知范围内的才返回 |
| 空间可用性 | `[(x, y, openness)...]` | 球场被网格化，每个格子的"空旷程度" |
| 自身状态 | `{stamina, fatigue, speed_cap, morale}` | 自身实时状态 |
| 战术指令 | `{role, zone, priority_target}` | 教练设定的战术角色 |

**视野属性的影响**：

- 视野90的球员：感知半径60m，扇角150°，能看到大半个球场
- 视野50的球员：感知半径25m，扇角80°，经常"没看到"更优传球线路
- 视野30的球员：基本只能看到身边的3-4个人，决策基于局部信息

**关键设计**：感知层返回的是**不确定信息**。视野外的球员位置是"估计值"，距离越远估计越不准。

### 第二层：思考层（球员"决定做什么"）

思考层分为**两个子决策**，按优先级排序：

#### 2A：情境决策（每0.5秒评估一次）

```
根节点：我在做什么？
├── 我方控球
│   ├── 我是持球人
│   │   ├── 前方有大空间？
│   │   │   ├── 盘带推进 [决策力影响：是否选择盘带vs传球]
│   │   │   └── 传给前方跑位的队友 [决策力影响：选择传给谁]
│   │   ├── 有射门机会？
│   │   │   ├── 射门！ [决策力影响：时机判断是否合理]
│   │   │   └── 再等等/再盘一步
│   │   ├── 被逼抢？
│   │   │   ├── 护球等待支援 [力量+盘带影响成功率]
│   │   │   ├── 传出安全球 [传球精度影响]
│   │   │   └── 试图突破盘带 [盘带+爆发力影响]
│   │   └── 无人逼抢+无压力
│   │       └── 持球观察，等待机会 [视野影响等待质量]
│   │
│   └── 我不是持球人
│       ├── 我在进攻三区？
│       │   ├── 跑位拉扯空间 [跑位意识+速度]
│       │   ├── 插上准备接应 [观察力]
│       │   └── 保持位置等反击机会
│       ├── 我是中场？
│       │   ├── 前插参与进攻 [攻击欲望属性]
│       │   ├── 回撤接应 [战术纪律]
│       │   └── 横向移动拉开宽度 [跑位意识]
│       └── 我是后卫？
│           ├── 留在防守位置 [战术纪律]
│           └── 参与进攻（边后卫套边）[战术指令]
│
├── 对方控球
│   ├── 球在我附近（<=5m）？
│   │   ├── 上抢逼抢 [防守侵略性+速度]
│   │   ├── 延缓等待队友回防 [防守意识]
│   │   └── 保持站位 [防守站位+位置感]
│   ├── 球在我防区？
│   │   ├── 回防到防守位置 [速度+体能]
│   │   ├── 盯防指定对手 [盯人能力+防守站位]
│   │   └── 保护空间/区域防守 [防守意识]
│   └── 球在远离我的区域
│       ├── 缓慢回撤到战术位置
│       └── 休息/慢跑恢复体力 [智能省力]
│
├── 死球/定位球
│   ├── 执行战术布置 [教练设定的定位球路线]
│   └── 回到默认站位
│
└── 球在空中
    ├── 我是门将→判断是否出击/站位
    ├── 我是防守人→判断落点并卡位
    └── 我是进攻人→判断落点并前插
```

#### 2B：微观动作决策（每帧/每tick执行）

```
每帧微观决策：
├── 盘带方向调整
│   ├── 目标方向
│   ├── 规避最近对手的方向
│   └── 当前朝向的平滑过渡 [盘带属性影响转身速度]
├── 是否加速
│   ├── 体力 > 30% → 可以加速
│   ├── 体力 < 15% → 只能慢跑
│   └── 爆发力决定加速的幅度和持续时间
├── 传球时机
│   ├── 队友跑到位了？[观察力+传球时机的匹配]
│   ├── 传球窗口是否打开？[对手是否封堵了传球路线]
│   └── 脚法选择：地滚球/直塞/过顶球/长传 [技术属性]
└── 射门时机
    ├── 与球门的角度和距离
    ├── 射门窗口（门将位置+后卫封堵角度）
    └── "冷静度"——越冷静越能等最佳时机
```

### 决策力属性对决策树的影响

**决策力不是简单地改变成功率，而是改变决策树中节点选择的"权重"**：

```
示例：前锋在禁区前沿持球，有3个选项

决策力95的前锋：
  射门权重: 30%（看到门将站位偏左，右脚抽射角度好）
  传球权重: 60%（看到右路队友已经跑出空位）
  盘带权重: 10%
  → 选择传球，助攻队友破门

决策力50的前锋：
  射门权重: 55%（"有机会就射"的简单策略）
  传球权重: 30%（看到了队友但不够确信）
  盘带权重: 15%
  → 选择射门，被后卫封堵

决策力25的前锋：
  射门权重: 70%（几乎不思考，本能射门）
  传球权重: 15%（注意不到队友跑位）
  盘带权重: 15%
  → 盲目射门，打飞
```

### 第三层：行动层（球员"怎么做"）

```
行动层输出：
├── 移动指令
│   ├── 目标点 (target_x, target_y)
│   ├── 移动速度 (max_speed × stamina_factor)
│   ├── 移动类型：冲刺/跑/慢跑/走/静止
│   └── 加速度（由爆发力决定）
├── 球操作指令
│   ├── 射门：方向、力度、脚法（推射/抽射/搓射）
│   ├── 传球：目标点、力度、类型（地滚/空中/直塞）
│   ├── 盘带：触球频率、方向
│   └── 停球：缓冲力度
└── 其他
    ├── 滑铲/抢断（时机、方向）
    ├── 头球（起跳时机、顶球方向）
    └── 大脚解围（方向、力度）
```

行动层的执行有**随机偏差**：

```
实际执行 = 指令值 + 高斯噪声(σ = f(技术属性, 疲劳度, 心理状态))

示例：
  传球指令：方向90°，力度70
  技术属性85，疲劳度低，心理稳定
  → 实际：方向92°，力度68（轻微偏差，球到位）

  传球指令：方向90°，力度70
  技术属性45，疲劳度高（80分钟），压力大
  → 实际：方向98°，力度60（明显偏差，球传大或传偏）
```

## 二、物理引擎

### 2.1 球的物理模型

```
球的运动方程（简化版）：

  position(t+dt) = position(t) + velocity(t) × dt
  velocity(t+dt) = velocity(t) + acceleration(t) × dt - friction × velocity(t) × dt

  其中：
  - friction：草皮摩擦系数（0.985/帧，球会自然减速）
  - acceleration：只在被"踢"时施加
  - 空中球：额外加入重力加速度 gz = -9.8m/s² 和空气阻力
```

**球的特殊物理**：

- **地滚 vs 弹跳 vs 空中**：三种状态有不同的摩擦/弹跳系数
- **旋转效果**：弧线球（香蕉球）通过在velocity中加入垂直分量模拟
- **弹射**：球碰到球员身体/门柱时产生反射，反射角度受碰撞面法线影响
- **草地纹理**：不同区域有微小的摩擦差异

### 2.2 球员移动物理

```
球员运动模型：

  当前速度 = min(目标速度, 最大速度 × 体力系数 × 加速状态)

  max_speed = 基础速度 × (1 - 疲劳度 × 0.4)

  转向：
  转向速率 = 基础转向速率 × 盘带系数
  转向时速度衰减 = 1 - 转向角度差 × 0.01（急转弯会减速）
```

**体力模型**（贯穿全场比赛的关键系统）：

```
体力消耗速率：
  走：0.1%/分钟
  慢跑：0.3%/分钟
  跑：0.6%/分钟
  冲刺：1.5%/分钟
  拼抢：1.0%/分钟
  无球跑动：0.4%/分钟

体力恢复速率：
  站立不动（非持球）：0.5%/分钟
  慢走：0.3%/分钟
  中场休息：恢复20-30%（取决于恢复属性）
  加时赛：恢复速率减半

体力阈值效果：
  100%-70%：全属性100%发挥
  70%-40%：速度-10%，失误率+50%，决策速度变慢
  40%-15%：速度-25%，失误率+150%，冲刺能力丧失
  15%-0%：  速度-50%，失误率+300%，可能抽筋
```

### 2.3 碰撞系统

```
碰撞检测分层：

Layer 1：AABB粗筛（每帧）
  → 将球场划分为网格（如2m×2m）
  → 只检查同一网格和相邻网格内的实体
  → 复杂度从 O(n²) 降到 O(n × k)

Layer 2：精确碰撞（仅通过粗筛的配对）
  → 球员-球员：圆柱体碰撞（半径≈0.5m）
  → 球员-球：点到圆柱距离检测
  → 球-门柱：球心到门柱中心距离 < 球半径+门柱半径
  → 球-边线：坐标越界检测

碰撞响应：
  球员-球员碰撞：
    → 轻微碰撞：双方减速，互相弹开
    → 冲刺碰撞（一方高速）：倒地动画，2秒恢复
    → 铲球碰撞：被铲球员倒地概率=铲球时机准确度
    → 犯规判定：碰撞点+力度+方向 → 裁判AI判定

  球员-球碰撞：
    → 持球检测：球员控制半径内 + 面朝球方向 ± 60°
    → 抢断检测：防守方在持球方控制半径内 → 抢断概率
    → 解围检测：后卫在危险区域碰到球 → 自动大脚
```

### 2.4 空间分区系统

```
感知网格（用于AI决策）：
  40m × 20m 球场 → 20 × 10 = 200个格子（每格2m×2m）

  每个格子存储：
  - 格子内有多少队友/对手
  - 格子是否是"空档"（openness = 1 - occupancy_factor）
  - 格子的"威胁值"（靠近球门 + 对手密集 = 高威胁）

战术网格（用于教练设定和团队AI）：
  40m × 20m → 10 × 5 = 50个格子（每格4m×4m）

  每个格子有一个权重值，表示"这个位置对当前战术有多重要"
  → 4-3-3阵型的权重分布 vs 4-4-2阵型的权重分布
  → 球员在无球时会被战术权重"吸引"，趋向于回到权重高的格子
```

## 三、团队AI协作

```
团队协调器（Team Coordinator）：
  运行频率：每2秒

  输入：当前比分、比赛时间、教练战术设定、球的位置和运动方向

  输出（广播给全队）：
  - 团队状态：全面进攻 / 攻守平衡 / 稳固防守 / 铁桶阵
  - 进攻方向偏好：左路主攻 / 右路主攻 / 中路渗透 / 均衡
  - 压迫线：全队上抢 / 半场压迫 / 回到本方半场

  每个球员收到团队指令后，在自己的决策树中增加权重
  → "团队状态=全面进攻" → 后卫插上权重+50%
  → "压迫线=半场压迫" → 前场逼抢权重+40%
```

**关键设计**：团队协调器的指令不是强制执行，而是**权重调整**。球员仍然会根据自身属性做出个性化决策。

## 四、比赛裁判AI

```
裁判AI：
  犯规判定：
    碰撞力度 > 阈值 + 碰撞方向不合理 → 犯规
    轻微犯规 → 任意球
    中度犯规 → 黄牌 + 任意球
    严重犯规 → 红牌
    最后一人犯规 → 红牌
    禁区内犯规 → 点球

  越位判定：
    传球瞬间，接球方身体最前端是否超过倒数第二名防守方球员

  门线技术：
    球完全越过门线 → 进球（像素级检测）
```

## 五、性能优化策略

```
1. 决策频率分层
   - 持球人：每帧决策
   - 球附近5m内的球员：每3帧
   - 其他球员：每10帧
   - 门将：每5帧

2. LOD（细节层次）
   - 球场中心区域：全精度物理+全精度碰撞
   - 远离球的区域：简化物理（线性插值）

3. 空间分区加速
   - 感知和碰撞都通过网格分区

4. 行为预测缓存
   - 非持球球员的"未来3秒预计位置"预计算

5. 异步AI更新
   - 渲染线程外，单独线程更新AI

6. 10倍速下
   - 物理步长增大
   - AI决策可跳帧
   - 实际计算量可能比1倍速还小
```

## 六、数据流全链路

```
一帧的计算流程：

[物理引擎] 更新球位置
    ↓
[感知系统] 每个球员更新感知数据
    ↓
[团队协调器] 每2秒更新一次团队指令
    ↓
[AI决策树] 每个球员做出决策
    ↓
[行动层] 决策转化为移动/球操作指令
    ↓
[物理引擎] 执行移动指令，更新位置
    ↓
[碰撞系统] 检测并处理碰撞
    ↓
[裁判AI] 判定犯规、越位等
    ↓
[渲染层] 绘制球场、球员、球
```

## 七、让AI表现"像人"的技巧

| 手法 | 效果 |
|------|------|
| **反应延迟** | 球员不会在球传到的瞬间做出反应，50-200ms反应时间 |
| **动作惯性** | 球员改变方向有过渡动画，不能瞬间转向 |
| **注意力不集中** | 低专注力的球员偶尔会"发呆"0.5-1秒 |
| **情绪波动** | 被犯规后1-2秒内情绪值上升，影响后续决策合理性 |
| **默契度偏差** | 传球方和接球方对"传球目标点"的理解有偏差 |
| **体能突变** | 体力在某些节点突然"断电"，模拟真实的体能瓶颈 |
