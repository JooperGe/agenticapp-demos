# AI FC — 实时决策与事件处理

> 头脑风暴第三轮：事件驱动+定时轮询双层架构、事件优先级、场景模拟、情绪系统、状态机

---

## 一、核心架构：双层决策系统

比赛中AI不能只靠"定时轮询"做决策，必须采用**事件驱动 + 定时轮询**的双层架构：

```
┌─────────────────────────────────────────────┐
│              事件总线 (Event Bus)             │
│   所有比赛事件实时广播，AI监听并即时响应         │
└──────────┬──────────────────────┬───────────┘
           │                      │
    ┌──────▼──────┐        ┌──────▼──────┐
    │  事件驱动层  │        │  定时轮询层  │
    │ (Interrupt) │        │  (Tick)     │
    │ 瞬时响应     │        │  周期性规划   │
    │ 优先级最高   │        │  背景决策     │
    └──────┬──────┘        └──────┬──────┘
           │                      │
           └──────────┬───────────┘
                      ▼
              ┌──────────────┐
              │   行动队列    │
              └──────┬───────┘
                     ▼
              ┌──────────────┐
              │   物理引擎    │
              └──────────────┘
```

---

## 二、事件分类与优先级

所有比赛事件分为5个优先级，高优先级事件可以**打断**低优先级决策：

| 优先级 | 事件类型 | 示例 | 响应方式 |
|--------|---------|------|---------|
| **P0 紧急** | 即时物理威胁 | 球飞向自己→门将、即将被撞、对方单刀直面门将 | **立即中断**，0帧延迟 |
| **P1 关键** | 比赛状态突变 | 丢球瞬间、进球、裁判吹哨 | **中断**，50ms反应延迟 |
| **P2 战术** | 教练指令/队友信号 | 教练喊"压上去"、队长手势、角球旗号 | **排队等待**，当前动作完成后立即执行 |
| **P3 机会** | 出现有利局面 | 发现空档、对手失误、反击机会 | 加入决策权重，下次轮询时优先考虑 |
| **P4 常规** | 周期性评估 | 位置调整、体力管理、跑位更新 | 正常定时轮询处理 |

## 三、实时事件处理流程

### 场景1：本方丢球的瞬间（攻转守）

```
事件触发：本方球员A传球被对手B截断
    │
    ▼
Event Bus 广播 [POSSESSION_LOST, culprit=A, interceptor=B, ball_pos]
    │
    ├──► 持球人A（丢球者）
    │     P1事件 → 立即中断"准备前插"动作
    │     → 反抢（防守侵略性高）/ 回追（战术纪律高）/ 停下来（疲劳度高）
    │
    ├──► A的队友（进攻群，距离球10-30m）
    │     P2事件 → 200ms内开始执行：
    │     → 距离球最近的1-2人：立即反抢
    │     → 中场球员：回撤到本方半场与进攻方之间
    │     → 边路球员：回防到对应边路
    │     → 前锋：根据战术指令决定是否参与逼抢
    │
    ├──► 后卫（已在防守位置）
    │     P4事件 → 收缩防线，盯防对应进攻球员
    │     → 门将：调整站位到球门偏左/偏右
    │
    └──► 团队协调器
          → 如果球在中场：全队半场压迫
          → 如果球在本方禁区：全线退守+密集防守
          → 如果球在对方半场：少数人逼抢
```

**真实感设计**：丢球后的反应不是"所有人同时转身"。前锋可能还在往前冲了0.5秒才意识到（低反应属性），后卫则已经收缩好了（高防守意识）。这种**时间差**就是真实感。

### 场景2：反击机会出现

```
事件触发：后卫C成功抢断，前方有大片空档
    │
    ▼
Event Bus 广播 [COUNTER_ATTACK_OPPORTUNITY, initiator=C, space_direction=right]
    │
    ├──► 抢断者C
    │     → 视野扫描（加速模式：0.1秒而非正常的0.3秒）
    │     → 找到前方跑位的队友？→ 立刻出球
    │     → 没找到？→ 自己带球推进
    │
    ├──► 前锋/边锋（反击第一梯队）
    │     跑位意识决定反应速度：
    │     → 跑位意识90：0.3秒内启动冲刺
    │     → 跑位意识50：0.8秒反应，犹豫要不要上去
    │     → 跑位意识20：1.5秒才意识到机会，来不及
    │
    └──► 团队协调器
          临时切换：[攻守平衡] → [全面进攻/快速反击]
```

**真实感设计**：反击能不能成功的关键不是"系统设定成功率"，而是多个球员的反应速度+跑位意识+速度属性的综合结果。

### 场景3：比赛中突发犯规/冲突

```
事件触发：对手D对己方球员E恶意犯规
    │
    ▼
Event Bus 广播 [FOUL_COMMITTED, fouler=D, victim=E, severity=high]
    │
    ├──► 裁判AI → 黄牌/红牌判定
    │
    ├──► 被犯规者E
    │     情绪系统触发：愤怒值 +40
    │     → 抗压低：冲向犯规者 → 可能吃第二张黄牌
    │     → 抗压中：倒地捂腿，但影响后续10分钟表现（失误率+30%）
    │     → 抗压高：冷静起身，不纠缠
    │
    ├──► E的队友（附近3-5人）
    │     → 团队协作高的队友：冲上去保护但克制
    │     → 侵略性高的队友：可能推搡 → 群体冲突风险
    │     → 队长：安抚双方
    │
    └──► 团队协调器
          己方被犯规：士气 +5（"憋着一口气"）
          己方犯规：情绪 -10（紧张感上升）
```

### 场景4：教练实时干预

```
用户操作：暂停 → 选择球员F → 下达指令"盯死对方10号"
    │
    ▼
Event Bus 广播 [COACH_INSTRUCTION, target=F, type=MARK_PLAYER, mark_target=opponent_10]
    │
    ├──► 球员F 进入"特殊盯人模式"：
    │     1. 更新目标位置（对方10号）
    │     2. 计算理想防守位置：目标与球门之间，距目标0.5-1.5m
    │     3. 移动到理想位置
    │     4. 目标进入自己防区 → 贴身逼抢
    │     5. 目标远离 → 放松跟随，保存体力
    │
    ├──► 其他队友
    │     F原来负责的区域出现空档 → 队内有人补位
    │
    └──► 教练UI
          球员F头顶出现"盯防10号"标记
```

## 四、球员状态机

每个AI球员在比赛中始终处于以下状态机中，事件可以触发状态跳转：

```
                    ┌──────────────┐
                    │   空闲/走位   │  ← 无球，回归战术位置
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │  跑位    │ │  回防    │ │  逼抢    │
        └────┬─────┘ └────┬─────┘ └────┬─────┘
             │            │            │
             │    ┌───────┴───────┐    │
             │    ▼               ▼    ▼
             │  ┌──────────┐ ┌──────────┐
             │  │  盯人    │ │  护球    │
             │  └──────────┘ └────┬─────┘
             │                     │
             │    ┌────────────────┘
             │    ▼
             │  ┌──────────┐
             │  │  持球    │ ← 获得球权
             │  └────┬─────┘
             │       │
        ┌────┼───────┼───────────────┐
        ▼    ▼       ▼               ▼
   ┌────────┐┌────────┐┌────────┐ ┌────────┐
   │  传球  ││  射门  ││  盘带  │ │  停球  │
   └────────┘└────────┘└────────┘ └────────┘

   特殊状态（事件触发跳转）：

   └────────┘  └────────┘  └────────┘  └────────┘
```

## 五、实时情绪系统

情绪不是视觉表现，而是**实时修改AI决策参数的隐藏乘数**：

```
情绪状态矩阵：

              低情绪值                高情绪值
              (消沉/沮丧)            (亢奋/激动)
                │                       │
   冷静 ← ─────┼───────────────────────┼────── → 躁动
                │                       │
   理性 ← ─────┼───────────────────────┼────── → 冲动
                │                       │
              消极                   积极

四个象限的效果：
┌─────────────┬─────────────────────────────┐
│  冷静+消极   │ 踢法保守，失误率+20%          │
│  (低迷期)    │ 不会冒险，但也不会创造机会     │
├─────────────┼─────────────────────────────┤
│  冷静+积极   │ 最佳状态：决策精准             │
│  (状态正佳)  │ 传球选择最优，失误率-15%       │
├─────────────┼─────────────────────────────┤
│  躁动+消极   │ 最差状态：决策混乱             │
│  (心态崩了)  │ 失误率+80%，容易犯规和冲突     │
├─────────────┼─────────────────────────────┤
│  躁动+积极   │ 高能状态：创造力+50%           │
│  (打鸡血)    │ 但失误率+30%，容易浪射        │
└─────────────┴─────────────────────────────┘

情绪实时更新公式：
  emotion = base_emotion + match_event_delta + time_decay + teammate_spread + coach_talk

  match_event_delta 事件影响：
    进球 → +25 / 失球 → -20 / 犯规被侵犯 → +5(冷静)-10(躁动)
    射门打偏 → -5 / 传球失误 → -3 / 成功抢断 → +8

  time_decay：情绪自然回归基准值（每分钟衰减5%）

  teammate_spread：队长领导力越高传播越快

  coach_talk 教练干预（中场休息）：
    → "激励" → 全队积极+15，躁动+10
    → "冷静" → 全队冷静+20，消沉+5
    → "战术分析" → 无情绪变化，决策力临时+10
```

## 六、每帧处理的具体流程

```
每帧（16ms@60fps，或33ms@30fps）：

Step 1: 物理引擎更新
  - 球位置更新
  - 球员位置更新
  - 碰撞检测与响应

Step 2: 事件检测与分发
  - 球权变化？ → 广播 POSSESSION_GAINED / POSSESSION_LOST
  - 进球？ → 广播 GOAL_SCORED
  - 犯规碰撞？ → 交给裁判AI判定
  - 越位？ → 裁判AI每帧检测
  - 教练指令？ → 从UI层收集并广播

Step 3: AI决策更新（分频执行）
  - 持球人：每帧 → 传球/射门/盘带/护球
  - 球附近5m内的球员：每3帧 → 逼抢/接应/回防
  - 其他球员：每10帧 → 跑位/保持位置/恢复体力

Step 4: 团队协调器（每120帧 = 2秒）
  - 读取比赛状态，评估威胁等级
  - 生成团队指令

Step 5: 行动执行
  - 将行动指令转化为物理参数
  - 加入属性偏差

Step 6: 情绪更新（每60帧 = 1秒）
  - 更新情绪值，计算队友传播

Step 7: 渲染
  - 绘制球场、球员、球、UI覆盖层
```

## 七、10倍速下的特殊处理

```
1. 物理步长调整
   - dt = 0.16s（每帧模拟10倍时间）
   - 用子步进：每帧内做4次 dt=0.04s 的物理计算

2. AI决策跳帧
   - 持球人：每帧 → 每3帧
   - 其他球员：每10帧 → 每20帧

3. 动画简化
   - 取消过渡动画，直接跳到目标位置
   - 远离球的区域用线性插值

4. 事件检测不变
   - 进球、犯规等仍然每帧检测
   - 反应延迟从100ms缩短到10ms

5. 实时暂停
   - 物理引擎停止，AI决策停止
   - 用户可下达教练指令
   - 恢复后从当前帧重新开始
```

## 八、一个完整的回合模拟

```
【比赛第67分钟，比分1:1，10倍速】

t=0.0s: 己方中场A在本方半场拿到球，边锋B沿边线冲刺
  A的决策力75 → 传球给B权重60%，盘带权重40% → 选择传球
  A的传球精度78 → 球偏了一点，B需要调整步伐接球

t=0.3s: B接球（停球技术82 → 停球质量良好）
  B面前后卫E距离12m，前方15m空档，中路队友F在禁区边缘
  B的决策力60 → 盘带推进权重45%，传中权重35% → 选择盘带

t=0.7s: B盘带推进10m，后卫E开始收缩逼抢
  B的决策重新评估 → 传中权重升为70% → 选择传中

t=0.9s: B传中（传中精度72 → 球飞向禁区）
  球预计0.8秒后落到F附近

t=1.0s: F感知到传中球，跑位意识85 → 立刻启动跑向落点
  对方后卫G晚启动0.15秒（反应延迟）
  F先到达落点前0.5m，G速度更快在追赶

t=1.7s: 球到达落点，F和G同时到达
  F弹跳78 vs G弹跳82，F身高180 vs G身高185
  但F先到位0.5m → 先触球概率+30%
  → F先顶到球，但头球精度70 → 角度偏差+8°
  → 球飞向球门右侧，但偏了

t=1.8s: 球出底线→球门球
  全队进入无球状态→回防位置→慢跑恢复体力

  → 教练可以此时暂停考虑：
     - B体力偏低，下半场换人？
     - F已经3次打偏，换替补前锋？
     - 中场休息对F做心理激励？
```

没有任何"脚本化"事件，所有行为都是球员属性+实时感知+决策树的有机结果。
