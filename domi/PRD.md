# 宅趣 Domi · 产品需求文档（PRD）

> 把家变成你喜欢的样子

- **产品名**：宅趣 Domi
- **定位**：空间灵感与软装规划工具
- **目标用户**：25-40 岁城市居住者 / 新家装修人群 / 租房改造爱好者
- **视觉调性**：温暖、干净、有呼吸感（奶油色系 + 陶土色点缀）
- **文档状态**：v0.1 头脑风暴草稿
- **更新日期**：2026-06-05

---

## 一、背景与机会

### 1.1 市场观察

- 小红书「家居改造」「租房布置」「软装搭配」年增长 300%+，用户习惯在社交平台找灵感，但「看到 → 买到 → 放好」的链路断裂
- 宜家 Place（AR 试摆）已停更，目前没有产品占据「拍照 → 识别 → 搭配建议 → 购买」的完整闭环
- 装修 App（住小帮、好好住）偏向硬装/长决策周期，缺乏「轻改造/软装搭配」的即时满足感
- AI 图像生成能力已经成熟（Stable Diffusion / Midjourney），但尚未被集成到「空间搭配」的垂直场景中

### 1.2 机会点

把「找灵感」的浏览行为与「改自己家」的行动意愿打通，用 AI 降低「试错成本」和「决策门槛」，让每个人都能像搭配衣服一样搭配自己的家。

---

## 二、产品定位

> 一个「发现 → 搭配 → 实现」的空间灵感 App。拍你的房间，AI 帮你换装。

### 2.1 核心差异

- **AI 空间试穿**：上传房间照片 → 选择风格/色调/单品 → AI 生成改造效果图
- **灵感 × 购物闭环**：每一个搭配方案都可直接购买（单品推荐 + 同款替代）
- **轻量、即时**：不碰硬装/施工，只做软装搭配与空间规划

### 2.2 不做什么

- 不做施工对接 / 装修公司撮合
- 不做户型图 / CAD 类工具
- 不做社交动态流（没有关注/feed）

## 三、用户画像

### 3.1 主画像 A：新家装扮者

- 28-35 岁，刚买房/租房，处于软装阶段
- 在好好住/小红书大量浏览，收藏夹爆满但无从下手
- 痛点：不知道哪些单品放在自己家好不好看、不敢买大件

### 3.2 主画像 B：租房改造爱好者

- 22-30 岁，租房但重视居住体验
- 预算有限，喜欢平价改造（宜家/闲鱼/1688）
- 痛点：房间不规则很难参考样板间、房东不允许大改动

### 3.3 主画像 C：家居内容浏览者

- 25-40 岁，日常刷家居内容但暂无改造需求
- 被美图吸引，种草后冲动消费
- 痛点：看到好看的不知道是什么牌子/哪里买

## 四、核心功能

### 4.1 灵感流（Inspire）

- 瀑布流卡片：风格标签（侘寂/北欧/中古/奶油/工业）+ 场景标签（客厅/卧室/角落）
- 每张卡片可「试穿到我家」：点击 → 选择到自己房间
- 长按单品识图 → 查看商品详情/购买链接

### 4.2 AI 空间试穿（核心）

```
【试穿旅程】
拍一张房间照片 / 从相册选择
    │
    ▼
自动识别空间区域（地面/墙面/家具轮廓）
    │
    ▼
选择改造目标：
  ├─ 换风格（"改成侘寂风"）
  ├─ 换单品（"换成绿色沙发"）
  ├─ 换色调（"暖色调"）
  └─ 自由描述（"加一盆龟背竹"）
    │
    ▼
AI 生成 2-3 个候选效果图
    │
    ▼
收藏方案 / 查看搭配清单 / 一键购买
```

### 4.3 搭配清单（Collage）

- 每个方案自动生成「搭配清单」：沙发 + 地毯 + 挂画 + 灯具 + 植物
- 每件商品显示：品牌 / 价格 / 购买链接 / 替代推荐（平价版 / 升级版）
- 可编辑清单（删/加/替换单品）
- 分享清单给朋友 / 导出

### 4.4 我的空间（Profile）

| 功能 | 说明 |
|------|------|
| 我的房间 | 已保存的房间 + 历史改造记录 |
| 风格画像 | 根据点赞/收藏/改造历史分析个人风格偏好 |
| 收藏夹 | 按风格/房间分组的收藏 |
| 改造日记 | 同一房间的 before/after 时间线 |

## 五、商业模式

| 层级 | 价格 | 权益 |
|------|------|------|
| 免费 | ¥0 | 每日 3 次 AI 试穿、基础灵感流浏览 |
| 灵感会员 | ¥19/月 · ¥168/年 | AI 试穿不限次、高级风格模型、去水印导出 |
| 搭配会员 | ¥49/月 | 含灵感会员全部 + 搭配清单商品折扣 9 折 + 专属搭配师 AI 对话 |

**额外收入**：
- 商品佣金（CPS）：搭配清单中商品购买链接的销售分成
- 品牌合作：风格标签冠名 / 精选搭配专题

## 六、技术选型（MVP）

- **前端**：Next.js + Tailwind（Web MVP）→ React Native（移动端）
- **AI 图像**：Stable Diffusion + ControlNet（空间区域保持）+ LoRA 风格微调
- **商品数据**：自建商品库（初期 5000+ SKU）+ 三方比价 API
- **识别**：Room segmentation（SAM / 自训练）+ CLIP 风格分类
- **存储**：PostgreSQL + 对象存储（图片）

## 七、关键指标

| 指标 | 目标 | 含义 |
|------|------|------|
| 试穿转化率 | > 15% | 浏览灵感 → 发起试穿 |
| 清单创建率 | > 25% | 试穿后生成搭配清单 |
| 商品点击率 | > 30% | 清单中点击商品详情 |
| 分享率 | > 10% | 分享清单/改造效果至社交平台 |
| 次周留存 | > 30% | 家居类产品基准线 |

## 八、冷启动

- **种子内容**：团队自摄 5 个典型房间改造（150 组 before/after），覆盖主流风格
- **合作渠道**：好好住 / 小红书 家居博主（置换体验会员）
- **传播钩子**：AI 改造效果对比图 → 天然适合社交分享
- **首发城市**：不限，纯线上
